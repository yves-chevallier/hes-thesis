\chapter{Interfaces graphiques}
Abordées au chapitre [refchapter], nous savons à présent que les interfaces graphiques seront composées d'un contenu en XHTML, d'une  présentation graphique en CSS et d'une librairie d'interactivité écrite en ECMAScript plus communément appelé JavaScript. Afin de mieux comprendre l'architecture d'une interface, ce chapitre est consacré à la réalisation d'une interface graphique la plus simple possible permettant le contrôle d'une seule lampe. 

Le GI aura pour dimensions 1024x768 pixels et disposera d'un contrôle de type SNVT\_switch pour la commande d'un éclairage LonWorks. Dans un premier temps, il faut concevoir la page dite statique. Imaginons un design avec une lampe dans le coin supérieur gauche, élégante manière pour indiquer qu'il s'agit d'une interface graphique de commande d'éclairage, d'un slogan amusant suivi d'une invitation à cliquez sur un bouton portant la mention "allumer" lorsque la lumière est éteinte et "éteindre" lorsque la lumière est allumée. La figure [reffigure] montre en (a) l'image utilisée lorsque la lampe est allumée et en (b) lorsqu'elle est éteinte.

\section{Comportement statique} 
Le code HTML nécessaire se résume, à peu près, à ceci: 


\begin{leftbar}\begin{verbatim}<html>
	<head>
		<link rel="stylesheet" href="touch.css" type="text/css">
	</head>
	<body>
		<div id="bouton" name="clickme" class="button"></div>  
	</body>
</html>\end{verbatim}
\end{leftbar}

On y voit que le document est embrassé de balises <html> et qu'il se compose de deux parties. L'une étant le préambule du document contenant certaines spécifications tels qu'ici, la balise <link/> étant un lien vers le fichier CSS utilisé. L'autre, le body, est le corps du document. Il contient un élément <div> qui deviendra par la suite le bouton. Il faut remarquer que le code HTML présenté ne répond pas aux spécifications XHTML1.0 STRICT. Pour satisfaire les exigences du w3c, il faut ajouter quelques lignes supplémentaires. Par la même occasion, un lien vers la librairie JavaScript décrite au chapitre [refchapter] est ajouté dans l'en-tête du code HTML:

\begin{leftbar}\begin{verbatim}<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
    "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
	<head>
		<title>Interface graphique de démonstration</title>
		<link rel="stylesheet" href="touch.css" type="text/css" />
		<script type="text/javascript" src="ltch.lib.js"></script>
	</head>
	<body>
		<div id="bouton" name="clickme" class="button"></div>  
	</body>
</html>\end{verbatim}
\end{leftbar}

A présent, il faut s'intéresser au  fichier CSS. Voici son contenu: 

\begin{leftbar}\begin{verbatim}.backimg_on 
{
	margin: 		0px;     	
	background-image: 	url(on.jpg);
	background-repeat: 	no-repeat; 	
}

.backimg_off 
{
	margin: 		0px;     
	background-image: 	url(off.jpg);
	background-repeat: 	no-repeat; 
}

.button 
{
	position: 		absolute;
	cursor: 			pointer;	
	left: 			650px; 
	top: 			470px; 
	width: 			210px; 
	height: 			35px; 
}\end{verbatim}
\end{leftbar} 

Le premier style déclaré correspond à celui appliqué au corps de la page lorsque la lampe est allumée. La commande margin force l'image à ne pas avoir de marges. La propriété background-image défini le lien vers l'image présentée sur la figure [reffigure]. Enfin, la propriété background-repeat permet d'indiquer au navigateur de ne pas considérer l'image comme un motif ainsi, de ne pas la répéter suivant l'axe horizontal et vertical. Le second style défini est identique au premier au détail près que le lien fait référence à l'image off.jpg. le dernier style défini les dimensions ainsi que la position du bouton. Le curseur de la souris prendra l'aspect d'une main lorsque celui-ci sera positionné dans l'espace du bouton. 

La figure [reffigure] présente l'aspect de la zone bouton en ayant ajouté la propriété background-color:red au style .button (b). En (a) on peut observer l'aspect de l'interface dans un navigateur web. 

\section{Comportement dynamique}
Le comportement dynamique sera décrit dans un fichier nommé touch.js il contiendra les lignes suivantes: 

\begin{leftbar}\begin{verbatim}controle1 = new SNVT_switch('bouton', 'lamp1')
controle1.Event('onstate', "document.body.className=(controle1.snvt.State())?'backimg_on':'backimg_off'")\end{verbatim}
\end{leftbar}

La première ligne peut s'expliquer facilement après la lecture du chapitre [refchapter]. La seconde relevant plus de l'utilisateur initié permet de modifier le style CSS appliqué à la balise <body> en fonction de l'état state du bouton. L'évènement onstate survient lorsque la valeur state est modifiée, soit après une mise à jour depuis le serveur, soit lors d'un clic de souris sur le bouton. 

\section{Fichier de description}
Dans sa version actuelle, le système requiert un fichier supplémentaire nommé touch.desc permettant de décrire l'interface lors d'une installation à l'aide du WEBGUI. Ce fichier de description se compose de trois parties. La première définit quelques paramètres de l'interface à savoir sa taille, son nom et une courte description. La seconde établit la liste des variables utilisées, leur type et leur direction. Enfin, la troisième partie est une liste des fichiers utilisés nécessaire pour que le GI soit opérationnel. 

\begin{leftbar}\begin{verbatim}# ./touch.desc: Description des variables d'interface
# 
[SETTINGS]
gi_size = 1024x768
gi_name = "Démonstrateur LonTouch no.2"
gi_desc = "Lumière c'est la vie"

# <id> <varname> <name>	<desc> <snvttype>	<direction>
[VARIABLES]
1	lamp1		"lumière 1"	""		SNVT_switch		OUTPUT 

# List of all file in GI
[FILES_LIST]
IMG   on.jpg
IMG   off.jpg
HTML	touch.html
JS 	touch.js
CSS	touch.css
DESC	touch.desc\end{verbatim}
\end{leftbar}    

\section{Arborescence des fichiers}
L'objet de cette section consiste en une simple énumération des fichiers nécessaires à la création d'une interface graphique. La version actuelle du WEBGUI et du démon ne supportent que le nom de touch pour les fichiers obligatoires.

\subsection{Fichiers obligatoires}
\begin{itemize}
\item touch.html
\item touch.css
\item touch.js
\item touch.desc
\end{itemize}

\subsection{Fichiers optionnels}
\begin{itemize}
\item images (*.jpg, *.jpeg, *.gif, *.png)
\item scripts additionnels (*.js)
\item ambiances sonores (*.mid, *.midi)
\item fichiers HTML additionnels (*.htm, *.html)
\end{itemize}
